Integration — 5 Steps
1

Copy the files

Drop the contexts/ and components/ folders into your Phase 1 src/ directory. Copy themes.js to src/themes.js.

2

Wrap your app with providers

In your main.jsx (or root component), wrap everything:

import { ThemeProvider } from './contexts/ThemeContext';
import { QueueProvider } from './contexts/QueueContext';

<ThemeProvider>
  <QueueProvider>
    <App />
  </QueueProvider>
</ThemeProvider>
3

Add ThemeSwitcher to your top bar

In your existing header/topbar JSX, import and drop in the component:

import ThemeSwitcher from './components/ThemeSwitcher';

// In your header JSX:
<div className="topbar">
  <span>NeonWave</span>
  <ThemeSwitcher /> // ← add this
</div>
4

Add Queue & Playlist panels + buttons

Add the panels at the root of your app's JSX, and buttons wherever your player controls are. See App.phase3.jsx for the QueuePlaylistButtons component you can drop in as-is.

import QueuePanel from './components/QueuePanel';
import PlaylistPanel from './components/PlaylistPanel';

// At the root of your returned JSX:
<>
  {/* ... your existing UI ... */}
  <QueuePanel />
  <PlaylistPanel currentTrack={currentTrack} />
</>
5

Connect track clicks to the queue

Replace your Phase 1 direct-play logic with loadQueue. This enables queue continuity, shuffle, and repeat to work correctly:

const { loadQueue, addToQueue } = useQueue();

// Play a track + rest of library:
const handleTrackClick = (track, index) => {
  loadQueue(allTracks, index);
};

// Get stream URL (use this for Howler):
const url = api.getStreamUrl(currentTrack.id);
const sound = new Howl({ src: [url], html5: true });
Add Theme-Aware Global CSS

Copy the GLOBAL_CSS string from App.phase3.jsx into your index.css. This adds scanline effects for Retro, grid lines for Synthwave, scrollbar theming, and smooth transitions:

ℹ
The CSS uses body[data-theme="retro"] selectors — themes.js automatically sets document.body.dataset.theme on every switch, so the effects are automatic.
